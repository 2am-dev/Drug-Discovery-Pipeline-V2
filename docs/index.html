Architecture

Seven-Stage Pipeline

Every inter-agent communication uses strict JSON validated against Pydantic schemas. No free-text is passed between stages.

🗂️
Phase 0

Planner

Builds the task dependency graph and estimates pipeline duration before any scientific work begins.

📚
Phase 1

Literature & Patent Retrieval

Mines PubMed (up to 150 abstracts), arXiv preprints, and PatentsView API. Embeddings stored in ChromaDB.

🎯
Phase 2

Hypothesis Generation

Selects a primary biological target with UniProt / PDB IDs, proposes a mechanistic hypothesis, and confidence-scores it.

⚗️
Phase 3

Molecule Design

Generates up to 20 candidate molecules via RDKit, filters on Lipinski rules, QED ≥ 0.5, and SA score ≤ 4.0.

🔬
Phase 4

Docking Evaluation

Runs AutoDock Vina on the top-5 shortlisted molecules, reporting ΔG (kcal/mol) and ligand efficiency.

🧪
Phase 5 optional

Synthesis Evaluation

RDKit SA scores plus LLM-proposed step-by-step synthetic routes for top-ranked molecules.

📄
Phase 6

Report Compilation

Assembles all agent outputs into a structured Markdown (or optional PDF) project proposal ready for review.

Features

What's Inside

A full-stack cheminformatics and AI stack running entirely on your own hardware — no external AI APIs required.

🤖

Smart Ollama Routing

Automatically tries your remote GPU server first, falls back to local Ollama within 5 seconds — zero manual switching.

✅

Strict JSON Validation

All inter-agent messages validated against Pydantic v2 schemas with up to 3 automatic retries on parse failures.

🧬

Real Cheminformatics

RDKit-powered molecule generation, Lipinski filtering, QED scoring, and synthetic accessibility checks.

📐

Token Budget Management

8,000-token prompt budget with HuggingFace tokenizer primary and tiktoken fallback. Auto-chunking for large literature.

🔌

External API Integration

PubMed ESearch/EFetch, arXiv, PatentsView, UniProt REST, and RCSB PDB — all wired up out of the box.

🐳

Docker Support

Dockerfile and docker-compose included for fully reproducible deployment across any environment.

Quick Start

Up and Running in Minutes

Clone, point to your Ollama endpoint, and run — that's it.

bash 
# Clone the repository
git clone https://github.com/2am-dev/Drug-Discovery-Pipeline-V2.git
cd Drug-Discovery-Pipeline-V2

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Copy and edit environment config
cp .env.example .env
# → Edit .env and set OLLAMA_REMOTE_URL to your server
bash 
# Basic run — disease indication
python main.py --indication "non-small cell lung cancer"

# Specific biological target with alternative model
python main.py --target EGFR --model gemma2:27b

# Full run with synthesis evaluation
python main.py --indication "Alzheimer's disease" --enable-synthesis

# Local Ollama only — skip remote server
python main.py --indication "KRAS-driven pancreatic cancer" --local-only

# Fast debug run — skip docking and patents
python main.py --indication "Type 2 diabetes" --no-docking --no-patents

# Verbose debug output
python main.py --indication "NSCLC" --log-level DEBUG
bash 
# ── Remote GPU server setup ───────────────────────
curl -fsSL https://ollama.ai/install.sh | sh

# Pull required models
ollama pull gemma4:31b-it-q8_0       # Primary LLM  (~20 GB)
ollama pull gemma4:26b-a4b-it-q8_0   # Alternative  (~16 GB)
ollama pull nomic-embed-text          # Embeddings   (~274 MB)

# Expose on network (trusted network / VPN only)
OLLAMA_HOST=0.0.0.0:11434 ollama serve

# ── .env configuration ───────────────────────────
OLLAMA_REMOTE_URL=http://192.168.1.100:11434/v1

# ── Local fallback (RTX 5070 Ti) ─────────────────
ollama serve   # Listens on localhost:11434 by default
bash 
# Build and start all services
docker compose up --build

# Run a single pipeline job inside the container
docker compose run pipeline \
  --indication "non-small cell lung cancer"

# Stop and remove containers
docker compose down
Connectivity

Automatic Failover

The pipeline health-checks your remote server first and silently falls back to local Ollama if unavailable — configurable timeout.

1 Remote Ollama  OLLAMA_REMOTE_URL  — tried first
│  timeout or connection refused (default 5 s)
▼
2 Local Ollama  localhost:11434  — automatic fallback
│  also unreachable
▼
✕ RuntimeError: No Ollama endpoint available — pipeline aborted
Performance

Token Budget

Operates within a strict 8,000-token prompt budget, leaving ~24,000 tokens of headroom for model reasoning inside Gemma4's 32K window.

32,000 token context window breakdown

Prompt budget8,000 tokens (25%)
Model reasoning headroom24,000 tokens (75%)
Input < 7,500 tokens
Pass directly to LLM
7,500 – 8,000 tokens
Trim least-critical fields
> 8,000 tokens
Summarise + compress
Configuration

Environment Variables

All settings controlled via .env or exported shell variables. Copy .env.example to get started.

Variable Default Description
OLLAMA_REMOTE_URL http://remote:11434/v1 Remote Ollama endpoint (tried first)
OLLAMA_LOCAL_URL http://localhost:11434/v1 Local Ollama fallback endpoint
OLLAMA_HEALTH_TIMEOUT 5 Health-check timeout in seconds
LLM_MODEL gemma4:31b-it-q8_0 Primary LLM model identifier
LLM_JSON_TEMPERATURE 0.2 Temperature for JSON-strict calls
MAX_CONTEXT_TOKENS 8000 Max tokens per LLM prompt
PUBMED_MAX_RESULTS 150 Max PubMed abstracts per run
MOL_GENERATION_COUNT 20 Molecules generated before filtering
MOL_SHORTLIST_COUNT 5 Top molecules passed to docking
VINA_EXHAUSTIVENESS 8 AutoDock Vina search depth
ENABLE_CHEMICAL_SYNTHESIS false Enable synthesis route evaluation phase
ENABLE_PDF_REPORT false Generate PDF alongside Markdown report
ENABLE_LLM_CACHE false Cache LLM responses to disk
