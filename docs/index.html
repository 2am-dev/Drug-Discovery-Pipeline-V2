Architecture

Seven-Stage Pipeline

Every inter-agent communication uses strict JSON validated against Pydantic v2 schemas. No free-text passes between stages.

🗂️
Phase 0

Planner

Constructs the full task dependency graph and estimates end-to-end pipeline duration before any scientific computation begins.

Task graph Dependency resolution ETA estimation
📚
Phase 1

Literature & Patent Retrieval

Mines PubMed (up to 150 abstracts), arXiv preprints, and the PatentsView API. Abstracts are batch-embedded (50 per chunk) into ChromaDB for semantic retrieval.

PubMed ESearch arXiv PatentsView ChromaDB nomic-embed-text
🎯
Phase 2

Hypothesis Generation

Selects the primary biological target, assigns UniProt and PDB IDs, identifies binding-site residues, proposes a mechanistic hypothesis, and scores confidence 0–1.

UniProt REST RCSB PDB Confidence scoring Alternative targets
⚗️
Phase 3

Molecule Design

Generates up to 20 de-novo SMILES, filters on Lipinski rule-of-five, QED ≥ 0.5, SA score ≤ 4.0, and shortlists the top 5 candidates for docking.

RDKit Lipinski filter QED scoring SA scoring Scaffold decoration
🔬
Phase 4

Docking Evaluation

Runs AutoDock Vina on the top-5 shortlisted molecules against the selected PDB receptor, reporting ΔG (kcal/mol), ligand efficiency, and key binding interactions.

AutoDock Vina ΔG prediction Ligand efficiency Pose export
🧪
Phase 5

Synthesis Evaluation Optional

Computes RDKit synthetic accessibility scores and leverages the LLM to propose step-by-step synthetic routes for the top-ranked molecules. Enabled via --enable-synthesis.

RDKit SA score LLM route planning Retrosynthesis hints
📄
Phase 6

Report Compilation

Assembles every agent's validated output into a structured Markdown project proposal. Optionally exports to PDF. Ready for scientific review with zero editing required.

Markdown report PDF export Full traceability
Features

Everything You Need

A production-grade cheminformatics and AI stack that runs entirely on your own hardware — no external AI APIs, no data leaving your network.

🤖

Smart Ollama Routing with Automatic Failover

Health-checks your remote GPU server first. Falls back to local Ollama within 5 seconds — completely transparent to the pipeline. Configurable timeout.

1
Remote Ollama  OLLAMA_REMOTE_URL  ← tried first
│  timeout / refused (5 s) → ▼
2
Local Ollama  localhost:11434  ← automatic fallback
│  also unreachable → ▼
✕
RuntimeError — pipeline aborted
📐

Token Budget Management

Strict 8K-token prompt budget. HuggingFace tokenizer primary, tiktoken fallback. Auto-chunks large literature into 50-abstract batches, compresses to ~500-token summaries.

Prompt budget8,000 tokens
Model reasoning24,000 tokens
Safety margin500 tokens
< 7,500 tokens Pass directly
7,500 – 8,000 Trim least-critical fields
> 8,000 tokens Summarise + compress
✅

Strict JSON Validation

All inter-agent messages validated against Pydantic v2 schemas. Up to 3 automatic retries with progressively stricter prompts on any parse failure.

🧬

Real Cheminformatics

RDKit-powered molecule generation, Lipinski filtering, QED scoring, SA scoring, and optional synthesis route evaluation. Not toy examples.

🔌

Six External API Integrations

PubMed ESearch/EFetch, arXiv, PatentsView, UniProt REST, RCSB PDB, and AutoDock Vina — all wired up and production-ready.

🐳

Docker Support

Dockerfile and docker-compose included for fully reproducible, isolated deployment on any machine or CI environment.

Quick Start

Up and Running in Minutes

Clone, point to your Ollama endpoint, and fire. The pipeline handles everything else automatically.

drug-discovery-pipeline — bash
# ── Clone and set up ────────────────────────────────────────
$ git clone https://github.com/2am-dev/Drug-Discovery-Pipeline-V2.git
$ cd Drug-Discovery-Pipeline-V2

# Create isolated virtual environment
$ python -m venv .venv
$ source .venv/bin/activate    # Windows: .venv\Scripts\activate

# Install all dependencies
$ pip install -r requirements.txt

# Configure environment
$ cp .env.example .env
# → Edit .env: set OLLAMA_REMOTE_URL to your server IP

# ── Run examples ────────────────────────────────────────────

# Basic — disease indication
$ python main.py --indication "non-small cell lung cancer"

# Specific biological target with model override
$ python main.py --target EGFR --model gemma2:27b

# Full run — synthesis evaluation included
$ python main.py --indication "Alzheimer's disease" --enable-synthesis

# Local Ollama only (skip remote server entirely)
$ python main.py --indication "KRAS-driven pancreatic cancer" --local-only

# Fast debug — skip docking and patent search
$ python main.py --indication "Type 2 diabetes" --no-docking --no-patents

# ── Remote GPU server setup ──────────────────────────────────
$ curl -fsSL https://ollama.ai/install.sh | sh

# Pull required models
$ ollama pull gemma4:31b-it-q8_0       # Primary LLM  (~20 GB)
$ ollama pull gemma4:26b-a4b-it-q8_0   # Alternative  (~16 GB)
$ ollama pull nomic-embed-text          # Embeddings   (~274 MB)

# Start server with network access (trusted network / VPN only)
$ OLLAMA_HOST=0.0.0.0:11434 ollama serve

# Set in .env on your client machine
OLLAMA_REMOTE_URL=http://192.168.1.100:11434/v1

# ── Local fallback (no remote server needed) ─────────────────
$ ollama serve    # Listens on localhost:11434 by default

# ── Docker deployment ────────────────────────────────────────

# Build and start all services
$ docker compose up --build

# Run a single pipeline job inside the container
$ docker compose run pipeline \
    --indication "non-small cell lung cancer" \
    --enable-synthesis

# Stop and remove all containers
$ docker compose down

# Rebuild image without cache (after dependency changes)
$ docker compose build --no-cache

# ── Debugging and diagnostics ────────────────────────────────

# Verbose debug output — raw LLM responses logged
$ python main.py --indication "NSCLC" --log-level DEBUG

# Pre-flight Ollama connectivity check
$ python scripts/check_ollama.py

# Check remote and local endpoints manually
$ curl http://your-remote:11434/api/tags
$ curl http://localhost:11434/api/tags

# Reset ChromaDB vector store
$ python scripts/clear_vectorstore.py

# Lower JSON temperature if getting parse errors
LLM_JSON_TEMPERATURE=0.1 python main.py --indication "NSCLC"

Configuration

Environment Variables

All settings controlled via .env or exported shell variables. Copy .env.example to get started.

Variable Default Description
OLLAMA_REMOTE_URL http://remote:11434/v1 Remote Ollama endpoint — tried first
OLLAMA_LOCAL_URL http://localhost:11434/v1 Local Ollama fallback endpoint
OLLAMA_HEALTH_TIMEOUT 5 Health-check timeout in seconds
LLM_MODEL gemma4:31b-it-q8_0 Primary LLM model identifier
LLM_JSON_TEMPERATURE 0.2 Temperature for JSON-strict calls
MAX_CONTEXT_TOKENS 8000 Max tokens per LLM prompt
PUBMED_MAX_RESULTS 150 Max PubMed abstracts per run
MOL_GENERATION_COUNT 20 Molecules generated before filtering
MOL_SHORTLIST_COUNT 5 Top molecules passed to docking
VINA_EXHAUSTIVENESS 8 AutoDock Vina search depth
ENABLE_CHEMICAL_SYNTHESIS false Enable synthesis route evaluation phase
ENABLE_PDF_REPORT false Generate PDF alongside Markdown report
ENABLE_LLM_CACHE false Cache LLM responses to disk
