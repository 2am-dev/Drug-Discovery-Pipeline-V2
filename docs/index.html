Hypothesis-to-Report Pipeline

An autonomous multi-agent system that accepts a disease indication or biological target as input and produces a comprehensive project proposal report — no human intervention required.

Architecture

Seven-Stage Pipeline

Every inter-agent communication uses strict JSON validated against Pydantic schemas. No free-text is passed between stages.

🗂️
PHASE 0

Planner

Builds the task dependency graph and estimates pipeline duration before any scientific work begins.

📚
PHASE 1

Literature & Patent Retrieval

Mines PubMed (up to 150 abstracts), arXiv preprints, and the PatentsView API. Results are embedded into ChromaDB.

🎯
PHASE 2

Hypothesis Generation

Selects a primary biological target with UniProt/PDB IDs, proposes a mechanistic hypothesis, and scores confidence.

⚗️
PHASE 3

Molecule Design

Generates up to 20 candidate molecules using RDKit, filters on Lipinski rules, QED ≥ 0.5, and SA score ≤ 4.0.

🔬
PHASE 4

Docking Evaluation

Runs AutoDock Vina on the top-5 shortlisted molecules against the selected PDB structure, reporting ΔG and ligand efficiency.

🧪
PHASE 5 (optional)

Synthesis Evaluation

Computes RDKit SA scores and proposes step-by-step synthetic routes for top-ranked molecules via LLM reasoning.

📄
PHASE 6

Report Compilation

Assembles all agent outputs into a structured Markdown (or PDF) project proposal report, ready for scientific review.

Features

What's Inside

A full-stack cheminformatics and AI stack, all running locally on your hardware.

🤖

Smart Ollama Routing

Automatically tries your remote GPU server first, falls back to local Ollama within 5 seconds — zero manual switching.

✅

Strict JSON Validation

All inter-agent messages are validated against Pydantic v2 schemas with up to 3 automatic retries on parse failures.

🧬

Real Cheminformatics

RDKit-powered molecule generation, Lipinski filtering, QED scoring, and synthetic accessibility checks — not toy examples.

📐

Token Budget Management

8,000-token prompt budget with HuggingFace tokenizer primary and tiktoken fallback. Automatic chunking for large literature sets.

🔌

External API Integration

PubMed ESearch/EFetch, arXiv, PatentsView, UniProt REST, and RCSB PDB — all wired up and ready to go.

🐳

Docker Support

Dockerfile and docker-compose included for reproducible deployment across any environment.

Quick Start

Up and Running in Minutes

Clone, configure your Ollama endpoint, and run. 

# 1. Clone and create virtual environment
git clone https://github.com/your-username/drug_discovery_pipeline.git
cd drug_discovery_pipeline
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 2. Configure environment
cp .env.example .env
# Edit .env → set OLLAMA_REMOTE_URL to your server IP
# Disease indication (basic)
python main.py --indication "non-small cell lung cancer"

# Specific target with alternative model
python main.py --target EGFR --model gemma2:27b

# Full run with synthesis evaluation enabled
python main.py --indication "Alzheimer's disease" --enable-synthesis

# Local Ollama only (skip remote server)
python main.py --indication "KRAS-driven pancreatic cancer" --local-only

# Fast debug run (no docking, no patents)
python main.py --indication "Type 2 diabetes" --no-docking --no-patents
# On your remote GPU server
curl -fsSL https://ollama.ai/install.sh | sh

# Pull required models
ollama pull gemma4:31b-it-q8_0       # Primary LLM  (~20 GB)
ollama pull gemma4:26b-a4b-it-q8_0   # Alternative  (~16 GB)
ollama pull nomic-embed-text          # Embeddings   (~274 MB)

# Start server with external access
OLLAMA_HOST=0.0.0.0:11434 ollama serve

# Set in .env
OLLAMA_REMOTE_URL=http://192.168.1.100:11434/v1
Connectivity

Automatic Failover

The pipeline tries your remote server first and silently falls back to local Ollama if the remote is unreachable.

1 Remote Ollama  OLLAMA_REMOTE_URL  ← tried first
│  timeout or connection error (5 s)
▼
2 Local Ollama  localhost:11434  ← automatic fallback
│  also unreachable
▼
3 RuntimeError raised — pipeline aborted
Configuration

Key Environment Variables

All settings are controlled via .env or exported environment variables.

Variable Default Description
OLLAMA_REMOTE_URL http://remote:11434/v1 Remote Ollama endpoint
LLM_MODEL gemma4:31b-it-q8_0 Primary LLM model
MAX_CONTEXT_TOKENS 8000 Max tokens per LLM prompt
LLM_JSON_TEMPERATURE 0.2 Temperature for JSON-strict calls
PUBMED_MAX_RESULTS 150 Max PubMed abstracts per run
MOL_GENERATION_COUNT 20 Molecules generated before filtering
MOL_SHORTLIST_COUNT 5 Top molecules passed to docking
VINA_EXHAUSTIVENESS 8 AutoDock Vina search depth
ENABLE_CHEMICAL_SYNTHESIS false Enable synthesis route evaluation
ENABLE_PDF_REPORT false Generate PDF alongside Markdown
